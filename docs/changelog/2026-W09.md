# Changelog — Week 9, 2026 (Feb 23 – Mar 1)

---

1. **Docs Structure Setup** (2026-02-28)
   Reorganized project documentation into `docs/` folder with structured planning workflow (`docs/plans/`). Merged `FINDINGS.md` into `docs/DISCOVERY.md`, then split into `docs/SCHEMA.md` (predicate inventories, class counts, data coverage) and `docs/DEBUGGING.md` (bugs #1–#8). Extracted conventions into `docs/CONVENTIONS.md`.

2. **Pipeline Restructure — Double Expansion, Wrong Anchors, Wrong Endpoint** (2026-02-28)
   Three interconnected pipeline bugs fixed:
   - **Double S2 expansion (near)**: Removed redundant `FILTER_S2_TO_REGION` step from all three relationship pipelines. Step 1 (`GET_S2_FOR_ANCHOR`) now filters to region directly in SPARQL, correcting the search radius from ~2-4km back to ~1-2km.
   - **All anchors shown regardless of proximity**: Added new `FILTER_ANCHOR_TO_NEARBY_TARGETS` step after `FIND_TARGET_ENTITIES`. If no targets found, pipeline exits early.
   - **Wrong endpoint for facility queries**: Changed `FIND_TARGET_ENTITIES` and `GET_ANCHOR_DETAILS` for facilities from `fiokg` → `federation`. The `kwg-ont:sfContains` triple requires the federated graph; fiokg alone returned ~14/42 Alabama facilities.

   *Files:* `planner.ts`, `executor.ts`, `templates/spatial.ts`, `templates/samples.ts`, `templates/facilities.ts`, `templates/waterBodies.ts`

3. **Distance Label Fix** (2026-02-28)
   Changed "~10km" to "~1-2km" in `RelationshipSelector.tsx` and `questionGenerator.ts` to accurately reflect the S2 Level 13 cell adjacency radius.

4. **Region Filter Before LIMIT Bug Fix** (2026-02-28)
   Fixed queries for small states (e.g. Alabama, 39 S2 cells) returning 0 results. `buildFacilityS2Query`, `buildSampleS2Query`, `buildWaterBodyS2Query` now accept `regionCode` and filter in SPARQL directly, eliminating the need for `LIMIT 5000` when a region is specified.

   *Files:* `templates/facilities.ts`, `templates/samples.ts`, `templates/waterBodies.ts`, `planner.ts`

5. **UI Refactor — Full-Width Map + Modal Editor** (2026-02-28)
   Removed sidebar layout; map is now full-width. `AnalysisQuestionBar` shows current question inline at top. Edit modal (`EditModal.tsx`) replaces the sidebar for editing queries. Modal shows `PipelineTimeline` in place of the editor when a query is running. Auto-closes 800ms after successful pipeline completion.

   *Files:* `Layout/`, `QueryEditor/EditModal.tsx`, `Pipeline/PipelineTimeline.tsx`, `store/queryStore.ts`

7. **Unified Flat Dropdown Component** (2026-02-28)
   Replaced all `react-select` and native `<select>` usages in the query editor with a new `FlatSelect` component that matches the `HierarchicalSelect` look and feel. Supports multi-select (checkboxes, dropdown stays open) and single-select (closes on pick), chip display with per-chip remove, portal rendering to escape modal clipping, and dynamic up/down positioning. Loading state added for async options (counties). All dropdowns now share consistent `.hs-*` styling.

   *Files:* `FlatSelect/FlatSelect.tsx` (new), `App.css`, `SampleFilters.tsx`, `WaterBodyFilters.tsx`, `RegionSelector.tsx`, `EntityTypeSelector.tsx`, `RelationshipSelector.tsx`

6. **Hierarchical NAICS Industry Dropdown** (2026-02-28)
   Replaced flat `react-select` dropdown in FacilityFilters with a custom `HierarchicalSelect` tree component. Industries are organized hierarchically by NAICS code prefix (2-digit → 3-digit → 4-digit → 5-6 digit). Selecting a parent auto-selects all children; only the parent shows as a chip. Dropdown renders via portal to escape modal clipping, with dynamic up/down positioning. Store now holds only user-selected codes (not expanded), with labels persisted for readable question text.

   *Files:* `HierarchicalSelect/useNaicsTree.ts`, `HierarchicalSelect/TreeNode.tsx`, `HierarchicalSelect/HierarchicalSelect.tsx`, `FacilityFilters.tsx`, `types/query.ts`, `utils/questionGenerator.ts`, `constants/prebuiltQueries.ts`, `App.css`

8. **Fix duplicate key warnings in FacilityLayer** (2026-02-28)
   Deduplicated facility features in `resultTransformer.ts` — same facility URI appeared in multiple SPARQL rows (one per industry code), causing React key warnings.

9. **Fix Edit button clearing map data** (2026-02-28)
   Removed `pipelineResult: null` from `openEditModal` in `queryStore.ts`. Map results are now preserved when the Edit modal opens; they only clear when a new pipeline runs.

10. **Disable Apply button when question is unchanged** (2026-02-28)
    Apply button in EditModal is now disabled when the current question matches the snapshot taken on modal open. Prevents unnecessary re-runs.
