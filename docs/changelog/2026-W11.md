# Changelog — Week 11, 2026 (Mar 9 – Mar 15)

---

1. **Query Documentation** (2026-03-10 – 2026-03-11)
   Added detailed query documentation for analysis questions, including step-by-step query details for `facilities-upstream-cumberland` and `samples-downstream-waste-indiana`.

2. **Wells Entity Type — Full Stack** (2026-03-12)
   Added wells as a fourth entity type alongside samples, facilities, and surface water bodies. Implemented end-to-end:
   - SPARQL prefixes for well data sources
   - `WellFilters` interface and `'wells'` added to `EntityType` and `MapLayer` unions
   - SPARQL query templates for well spatial queries
   - Pipeline wiring in planner, executor, and result transformer
   - `WellFilters` and `WellLayer` components
   - Integrated wells into `EntityTypeSelector`, entity block display, and map rendering

   *Files:* `constants/prefixes.ts`, `types/query.ts`, `types/map.ts`, `engine/templates/wells.ts` (new), `engine/planner.ts`, `engine/executor.ts`, `engine/resultTransformer.ts`, `components/QueryEditor/WellFilters.tsx` (new), `components/Map/WellLayer.tsx` (new), and related UI components

3. **Fix Water Body Feature Type Filter** (2026-03-12)
   The `VALUES` clause for `ftype` filtering in water body queries had no effect because `?ftype` was never bound. Added the missing `nhdplusv2:hasFTYPE` triple to `buildWaterBodyFilterQuery` and `buildWaterBodyS2Query`.

   *Files:* `engine/templates/waterBodies.ts`

4. **Rename "Water Bodies" → "Surface Water Bodies"** (2026-03-12)
   Updated labels, descriptions, and prebuilt query tags to distinguish surface water bodies from the new wells entity type.

   *Files:* `components/QueryEditor/WaterBodyFilters.tsx`, `constants/prebuiltQueries.ts`

5. **URL-Based Routing** (2026-03-12)
   Replaced Zustand view state (`AppView`, `currentView`, `navigateTo`) with React Router nested routes. Dashboard lives at `/`; each analysis question gets its own URL at `/q/:queryId`. Navigation now uses `useNavigate()` and `useLocation()`. Removes `AppView` enum and navigation actions from the store entirely.

   *Files:* `main.tsx`, `App.tsx`, `store/queryStore.ts`, `components/Layout/EditorView.tsx` (new), `components/Layout/Header.tsx`, `components/Dashboard/Dashboard.tsx`, `App.css`

6. **Design System Page** (2026-03-12)
   Added a `DesignSystem` component at `/design-guide` that showcases reusable UI components and design token values.

   *Files:* `components/DesignSystem/DesignSystem.tsx` (new), `components/DesignSystem/DesignSystem.css` (new)

7. **Fix Netlify SPA Routing** (2026-03-12)
   Added `public/_redirects` with a catch-all rule so direct URL access to routes like `/q/:queryId` is served by `index.html` rather than returning a 404.

   *Files:* `public/_redirects` (new)

8. **Fix Ghost Points in Map Export** (2026-03-13)
   Replaced `html2canvas` with `modern-screenshot` to fix exported JPGs showing sample markers displaced northward. The root cause was html2canvas's inability to handle Leaflet's `translate3d` CSS transforms — SVG circle positions were baked into serialized data URLs without the transform offset. `modern-screenshot` handles all CSS transform functions natively, eliminating the issue without any `onclone` workaround.

   *Files:* `package.json`, `src/utils/exportMap.ts`
